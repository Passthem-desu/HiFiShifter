# HiFiShifter 使用手册（中文）

本文档面向最终用户，介绍 HiFiShifter 的基本使用方式与常见操作。

## 1. 快速上手

- 打开应用后，界面主要由三部分组成：
  - **时间线（Timeline）**：编排音频剪辑、轨道与播放头。
  - **钢琴卷帘/参数面板（Piano Roll / Params）**：编辑音高（Pitch）等参数曲线。
  - **顶部菜单/工具栏与底部状态栏**：
    - 顶部两条（菜单栏 + 工具栏）用于工程操作、工具/参数选择、播放控制等。
    - 底部状态栏用于显示运行状态与提示信息。

## 2. 工程（Project）

- **新建工程**：菜单 `File → New`（不同构建版本菜单项可能略有差异）。
- **打开工程**：菜单 `File → Open`。
- **保存工程**：菜单 `File → Save`；另存为使用 `File → Save As`。

> 说明：在 Tauri/Rust 模式下，工程状态（包括撤销/重做栈）以“后端为权威”，因此保存/撤销行为会更稳定。

## 3. 导入音频

- **导入音频文件**：
  - 使用菜单导入（若有）：`File → Import Audio`。
  - 或将音频文件拖拽到时间线区域（取决于当前版本是否启用拖拽导入）。

导入后会在目标轨道上生成一个剪辑（Clip）。

## 4. 时间线基础操作

### 4.1 播放/停止与跳转

- **播放/停止**：使用界面按钮或菜单（具体取决于版本）。
- **提示：启用 Pitch Edit 时的“渲染中/短暂无声”**：当工程启用了 Pitch Edit（例如 WORLD/NSF-HiFiGAN）后，点击播放可能会先短暂无声并在状态栏显示“渲染中...”，待后端预缓冲完成后再开始出声（用于避免“原声 → 变调”的突变与回调卡顿）。
- **跳转（Seek）**：在时间线标尺或空白区域点击以移动播放头。

### 4.2 选择（Selection）

- **单选剪辑**：左键点击剪辑。
- **多选剪辑**：右键按住并拖拽进行框选（要求版本启用框选交互）。

### 4.3 剪辑编辑

- **移动剪辑**：拖拽剪辑。
- **分割剪辑（Split）**：选中剪辑后按 `S`，在播放头位置分割为两段。
- **胶合剪辑（Glue）**：同轨道上多选至少 2 个剪辑后，使用右键菜单选择“胶合”。
- **裁剪（Trim）**：拖拽剪辑左右边界，调整剪辑的入点/出点（不改变源文件）。
- **淡入淡出（Fade In/Out）**：
  - 可通过剪辑属性面板输入淡入淡出长度；或在剪辑边缘手柄拖拽（取决于版本）。
- **变速/伸缩（Playback Rate / Time Stretch）**：
  - 通过剪辑属性面板调整 `Rate`；部分版本支持 `Alt + 拖动边界` 进入伸缩模式。

### 4.4 轨道操作

- **静音/独奏**：在轨道条目上切换 Mute/Solo。
- **轨道重排与嵌套**：在左侧轨道列表拖拽轨道条目：
  - 上下拖动：同级内重排
  - 向右拖入：作为另一轨道的子轨道（嵌套）

### 4.5 撤销/重做

- **撤销**：`Ctrl + Z`
- **重做**：`Ctrl + Y`

## 5. 参数编辑（Pitch / Tension 等）

- 在底部参数面板选择要编辑的参数（例如 Pitch）。
- 在钢琴卷帘/参数区域进行绘制或编辑（具体交互随版本演进）。

> 提示：参数曲线属于工程内容，保存工程后会随工程一起保存。

## 6. 波形显示与缓存

- 时间线会按当前缩放请求波形 peaks，用于显示更清晰的波形预览。
- 若波形显示异常或磁盘缓存占用过大，可使用菜单 `视图 → 清除波形缓存`。

## 7. 导出

- 使用菜单导出（若有）：`File → Export` / `Export Audio`。
- 导出通常会渲染时间线混音结果为 WAV。

## 8. 常见问题

- **看不到波形/波形很细**：尝试放大时间线；或执行“清除波形缓存”后重启再试。
- **按下播放无声音**：确认轨道未静音、剪辑未静音；并检查系统默认输出设备。
- **撤销/重做行为奇怪**：优先在 Tauri/Rust 模式下使用（后端权威撤销栈）。

## 9. 高级排障（可选）

本节面向需要定位性能/卡顿问题的高级用户。

- **实时播放卡顿/无声排查（开发模式）**：
  - 可设置环境变量 `HIFISHIFTER_DEBUG_RENDER_STATS=1` 启用后端的实时渲染统计。
  - 统计信息用于区分：是 pitch hard-start 在等待覆盖（后端渲染跟不上），还是 base_stream 欠供导致回退到实时混音。
  - 开发/调试可通过命令 `debugRealtimeRenderStats` 获取计数结果（该命令属于调试用途，不影响正常功能）。
